### CG概論 榊原
AOV
>> レンダリング時にディフューズ（固有色）、スペキュラ（ハイライト）、Z深度（奥行き）などの要素を個別の画像として出力する機能のこと、コンポジッターが合成作業をする際に、各要素を微調整するときに使われる
> スケール
>> モデルや空間の大きさのこと。テクスチャやエフェクトつける際は、最も意識するべき要素である

解像度 (アスペクト比)
> pixcel sample
>> 一ピクセルをさらに3*3に分割し、それを１ピクセルに置き換える(この処理は重くなる)

カメラ
> parspective
>>　とったカメラの見た目、パースを維持して描写する、3D
> ousographic
>> とったカメラのパースを無視して描写する、2D

> FOV
>> 焦点距離、被写界深度のこと。

> 画角 (Focal Length)
>> 映る範囲の角度
>> (広角)18,24,35,50(標準),85,135,200(望遠)
> 被写界深度 (Apperture)
>> カメラでいうところのピント
> 焦点距離 (Focal Distance)
>> 焦点距離
> Shutter

ポリゴン
> オブジェクトを構成する頂点座標を結んだ面
>> ゲームでは主に三角形ポリゴン(unrialengineが三角形ポリゴン描画のため)

PBR(PhysicalBacedRendering)
> Albedo
>> 正しい色
> Nomal
>> 凹凸
> Raughness
>> 表面の質感(粗さ)
> Metalic
>> 金属質

> bloom
>> 放射の表現
> motionblur
>> オブジェクトの残像表現
> DOF(depth of field) 
>> カメラが鮮明に見える範囲
> ambient occulusion 
>> 接触面に発生する暗さの強調表現
> colorgrading

AOV
> レンダリングした情報を分割して管理することができる手法
>> bearuty : 普段レンダリングして出てくるもの。すべての情報が構造化されて出力されている

ライティング
> 三点照明
>> CG制作において背景などにキャラクターをなじませ、かつ際立たせるライティング手法
> 環境光(HDRI)
>> 天球に貼り付けることによって、環境光を演出する

レンダリング
> リアルタイムレンダリング
>> 主にGPUを使ってレンダリングする(主にゲームで使われる)
>>> まずCPUがシーンを構築する(現在の環境の読み込み)、その後GPUが描画指示を出す(カリング処理:カメラの外側や建物の裏側などは描画指示を出さない設定にすることで軽量化する)。そうすることで頂点の座標を取得し、それをもとにポリゴンが生成される。その後ラスタライズ(映像として描画できる形に変換)し、それにポストプロセス(質感表現)がなされたものをレンダリングする
>>> 上記工程が毎フレーム行われる

> プリレンダリング
>> 主にCPUを使ってレンダリングする(主に映像で使われる)
>>> cameraからrayを飛ばし、そのrayがcameraに帰ってくるまでを計算することで描画を確定する
>>> 交差、material、light、間接光


> レンダリング手法
>> レイトレーシング(RayTracing)
>>> 光を飛ばしてその反射率を計算してレンダリングする手法
>> パストレーシング(PathTracing)

>> ラスタライゼーション
>>> Raytracingを切った時のライティングの情報を色として表現する。Raytracingよりも軽量であるため、ゲームで多く使われている手法

>> Scanline 
>>> 光の屈折率を表現できない分軽量
>> Toon
>>> アニメーションやセルルックに表現する手法
>> Rudiocity
>> ZBuffer

> ゲームにおけるレンダリング

レンダラー
> RenderMan
>> リアルタイムでレンダリング後の絵を見ながらレンダリングの設定を変更することができる

> Lumen
>> mayaなどの標準レンダラーよりもきれいでかつ高速なレンダラー

構図
> 業界ではフレーミングという

ウェイト
> オブジェクト同士の接触を判定し、貫通しないように調整すること
> クロスシュミレーションなど

リトポロジー
> zbrushなどで作ったハイポリゴンオブジェクトをゲームなどで動くような頂点数にすること
> 人間の顔などをリトポロジするときは、耳の周りや、目、口などを軸に同心円状にループを作る(筋肉の構造のように)つけるのが望ましい

おまけ
> メモリ削減のために面が閉じていないモデルを作ってしまうと、エフェクトをつけようとする際に問題が生じる。
> 先生のおすすめするレンダラー: RenderMan(blenderでも使える。)( ピクサー(現在のCGの大本をほとんど作った)が作ったレンダラー)
> 時短できるならありものを加工して作ってもいい
